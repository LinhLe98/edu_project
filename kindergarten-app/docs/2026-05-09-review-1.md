# 2026-05-09 — Review Iteration 1

## Scope
UI consistency, UX clarity, code structure review + frontend-design validation.

## Issues Found & Fixed

### 1. SectionHeadingComponent — no left-align support
**Problem**: Component forced `text-center` on all uses, but SchoolOverviewComponent needed left-aligned heading in a 2-column layout. The workaround was a hacky `class="text-left [&>div]:text-left ..."` override.
**Fix**: Added `@Input() align: 'center' | 'left' = 'center'` to SectionHeadingComponent. Accent bar and subtitle width/margin now respect the align setting. Used `align="left"` in SchoolOverviewComponent.

### 2. TeacherHighlightsComponent — 5 cards in 4-column grid
**Problem**: `staffService.getFeatured()` returned 5 members. With `lg:grid-cols-4`, this created a lone orphan card on the second row.
**Fix**: Added `.slice(0, 4)` to limit the home page teacher highlights to exactly 4 cards.

### 3. FacilitiesSectionComponent — double SectionHeadingComponent
**Problem**: A full `SectionHeadingComponent` was used for the minor sub-header "Hình Ảnh Cơ Sở", creating visual noise and excessive vertical spacing before the photo grid.
**Fix**: Replaced with a simple `<h3>` tag.

### 4. Unused app.html (default Angular CLI template)
**Problem**: Angular CLI generated `src/app/app.html` (20KB default welcome page) which was never referenced since `app.ts` uses an inline template.
**Fix**: Deleted the file.

## Frontend-Design Validation Results
✅ Clean layout — no orphan elements after 4-card fix

✅ Navigation — `routerLinkActive` highlights active route in header, mobile drawer shows active peach background
