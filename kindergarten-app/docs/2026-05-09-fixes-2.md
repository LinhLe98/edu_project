# 2026-05-09 — Bug Fixes: Home News Navigation & Learning Materials

## Summary
Two user-reported bugs fixed. All changes pass `ng build` with zero errors.

---

## Bug Fixes

### 1. News on Home page — clicking a card did nothing

**Root cause:** `latest-news.component.html` rendered each `<app-card-news>` without any
`[routerLink]` wrapper. The `news-list` page already used the correct pattern (each card
wrapped in `<a [routerLink]="['/tin-tuc', article.slug]">`), but the home page section did not.

**Fix:**
- Added `RouterLink` to `latest-news.component.ts` imports array.
- Wrapped each `<app-card-news>` in `<a [routerLink]="['/tin-tuc', article.slug]" class="block h-full no-underline">` in the template, matching the pattern used by `news-list.component.html`.

Files changed:
- `src/app/pages/home/latest-news/latest-news.component.ts`
- `src/app/pages/home/latest-news/latest-news.component.html`

### 2. Learning materials — "document not found" error on view/download

**Root cause:** All 9 materials in `materials.data.ts` pointed to
`https://www.w3.org/WAI/WCAG21/Techniques/pdf/PDF2.pdf`, which returns HTTP 404
(the document was removed from the W3C server). Both `viewMaterial` (window.open)
and `downloadMaterial` (fetch + blob) therefore failed to load any file.

**Fix:** Replaced all 9 `fileUrl` values with
`https://www.africau.edu/images/default/sample.pdf`, a stable public sample PDF
widely used for UI testing. This is mock data — a single test PDF for all materials
is appropriate for a demo site.

File changed:
- `src/app/core/data/materials.data.ts`

uses `RouterLink`).

Added 2 new tests (total: 5 tests):

  fails if any material still uses the old 404 URL.
