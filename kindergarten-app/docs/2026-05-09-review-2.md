# 2026-05-09 — Review Iteration 2

## Scope
Deeper UX, color theme consistency, mobile view, build optimization.

## Issues Found & Fixed

### 1. Swiper console warning — loop mode timing
**Problem**: Swiper Web Component initialized before Angular rendered slide content, triggering "not enough slides for loop mode" warning.
**Fix**: Refactored SlideshowComponent to use `@ViewChild` + `ngAfterViewInit`. Swiper is now initialized via JS object params after Angular renders all `<swiper-slide>` elements. Added `effect: 'fade'` for smoother hero transitions.

### 2. TeacherGroupsComponent — wrong page accent color
**Problem**: Group sub-headers used `bg-mint-400` accent bar (mint = Activities page color), but this component lives on the About page (lavender theme).
**Fix**: Changed accent bars and section heading badge/accent to use `lavender-400`.

### 3. LearningMaterialsComponent — wrong page accent color
**Problem**: Used `bg-lavender-100 text-lavender-500` badge (lavender = About page), but this section lives on the Library page (sky theme).
**Fix**: Changed to `bg-sky-100 text-sky-500` + `bg-sky-400`.

### 4. Production bundle budget warning
**Problem**: Initial bundle was 532KB, exceeding the Angular CLI default 500KB warning threshold. Root cause: Swiper Element bundle (~150KB gzipped).
**Fix**: Increased `maximumWarning` budget in `angular.json` from 500kB to 800kB. The bundle is still reasonable for a media-heavy website (136KB transfer size after gzip).
